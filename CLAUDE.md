# CLAUDE.md

이 파일은 Claude Code가 이 저장소에서 작업할 때 따르는 기본 규칙을 정의합니다.

## 프로젝트 개요

모의면접 매칭 서비스의 **소개(랜딩) 페이지**입니다.

- **누구를 위해**: 모의면접을 구하는 사람
- **무엇을**: 같은 니즈를 가진 사람들을 서로 찾을 수 있게 한다
- **왜**: 오프라인에서 미팅 장소를 잡거나 사이트 내 화상 면접 기능으로 면접을 진행함으로써,
  누군가 앞에서 면접을 보며 **타인에게 자신의 생각을 표현하는 능력**을 기르도록 돕는다

이 저장소는 서비스 자체가 아니라 서비스를 소개하는 **단일 랜딩 페이지**를 만든다.
화상 면접·매칭 같은 기능은 "소개 대상"일 뿐, 이 페이지에서 실제로 구현하지 않는다.

## 기술 스택 및 제약

- **React + TypeScript + Vite** 기반의 정적 사이트.
- 스타일링은 **Tailwind CSS**로 한다. (전역 리셋 외의 커스텀 CSS는 최소화)
- **DB 없음**, 백엔드 없음. `vite build` 결과물(`dist/`)이 정적 호스팅만으로 동작해야 한다.
- 외부 의존성은 최소화한다. 새 패키지 추가는 꼭 필요한 경우에만, 추가 전에 이유를 먼저 설명한다.
- 라우팅이 필요해지면 단일 페이지 스크롤 우선으로 두고, 굳이 라우터를 도입하지 않는다.

## 실행 명령어

- `npm run dev` — 개발 서버 (Vite, HMR)
- `npm run build` — 프로덕션 빌드 (`dist/`)
- `npm run preview` — 빌드 결과 로컬 미리보기
- (설정 시) `npm run lint` — ESLint 검사

## 파일 구조

```
landing-service/
├── index.html              # Vite 엔트리
├── src/
│   ├── main.tsx            # 앱 진입점
│   ├── App.tsx             # 페이지 조립
│   ├── components/         # 섹션·UI 컴포넌트 (Hero, Features, CTA 등)
│   ├── assets/             # 이미지 등 에셋
│   └── index.css           # Tailwind 지시자 + 전역 스타일
├── tailwind.config.js
├── vite.config.ts
├── tsconfig.json
└── CLAUDE.md
```

- 섹션 단위로 컴포넌트를 분리한다 (`Hero.tsx`, `Features.tsx`, `HowItWorks.tsx`, `CTA.tsx` 등).
- 컴포넌트 파일명은 PascalCase, 그 외 일반 모듈은 camelCase.
- 새 파일을 만들기 전에 기존 컴포넌트에 넣는 게 자연스러운지 먼저 검토한다.

## 코딩 규칙

- **함수형 컴포넌트 + 훅**만 사용한다. 클래스 컴포넌트 금지.
- props는 `interface`/`type`으로 명시한다. `any`는 피한다.
- 스타일은 JSX의 `className`에 Tailwind 유틸리티로 작성한다. 인라인 `style`은 동적 값에 한해서만.
- 반복되는 Tailwind 클래스 묶음은 컴포넌트로 추출한다.
- 색상·간격·폰트 등 디자인 토큰은 `tailwind.config.js`의 `theme.extend`에 정의해 재사용한다.
- HTML 시맨틱을 지킨다(`header`, `nav`, `section`, `footer`). 한글 깨짐 방지를 위해 `lang="ko"`, `<meta charset="UTF-8">` 유지.
- 접근성: 이미지 `alt`, 인터랙티브 요소의 키보드 접근과 충분한 대비를 보장한다.
- 모바일 우선 반응형: 기본 스타일은 모바일, `sm:`/`md:`/`lg:`로 확장한다.

## 콘텐츠·디자인 원칙

- 첫 화면(Hero)에서 **"무엇을 하는 서비스인지"**가 한눈에 들어와야 한다.
- 위 "누구/무엇을/왜"를 핵심 메시지로 구성한다: 대상 → 핵심 가치 → 동작 방식 → 행동 유도(CTA).
- 과한 애니메이션·자동재생을 피하고, 빠른 로딩과 가독성을 우선한다.
- 카피는 한국어로 작성한다.

## 협업 규칙

- 사용자와의 대화 및 코드 주석은 **한국어**로 한다.
- 변경 후에는 무엇을 왜 바꿨는지 간단히 설명한다.
- 추측이 필요한 결정(스택 변경, 새 의존성 추가 등)은 진행 전에 먼저 확인한다.
